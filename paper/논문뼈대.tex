\section{핵심 스토리 (한 문단)}
% ============================================================

슬로싱 실무자(탱크 검사원, 구조 엔지니어)는 CFD 전문 지식 없이는 시뮬레이션을 수행할 수 없다.
기존 LLM-CFD 에이전트 7종은 모두 \textbf{CFD 전문가 대상 + OpenFOAM(격자) + 클라우드 API}다.
본 연구는 \textbf{비전문가가 자연어로 SPH 슬로싱 시뮬레이션을 자율 수행}할 수 있는 최초의 AI 에이전트 SlosimAgent를 제시한다.
10개 시나리오에서 M-A3 파라미터 정확도 61\%를 달성하고(32B$\approx$8B, $\Delta$=2.5\%pp),
2$\times$2 factorial ablation으로 도메인 프롬프트(+65.3\%pp)와 도구(+6.9\%pp)의 기여를 분리하며,
SPHERIC Test 10 벤치마크로 물리적 신뢰성을 정량 검증한다(M2=19.5\%, r=0.655).

\subsection{4. Evaluation ({\small\color{gray}$\sim$4p})}

\textbf{4.1 NL$\rightarrow$XML Parameter Fidelity (EXP-A, {\small\color{gray}$\sim$2p})}
\begin{itemize}[nosep,leftmargin=1.5em]
\item 실험 설정: 10개 시나리오 $\times$ 2 모델(32B, 8B) $\times$ 3 trials = 60 runs, temperature=0
\item M-A3 8-parameter 채점: tank$_{xyz}$, fill\_height, motion\_type, frequency, amplitude, timemax
\item \textbf{결과: 32B 61.2\% $\approx$ 8B 58.7\% ($\Delta$=2.5\%pp) --- 모델 크기 무관}
\item 난이도별: Easy 87.5\%, Medium 69.8\%, Hard 23.3\% (원통형/STL 미지원)
\item Determinism: 32B 10/10, 8B 9/10 시나리오에서 M-A3 $\sigma$=0.0 (완전 재현)
\item 체계적 오류 패턴: P1 motion\_type 혼동(5/10), P2 amplitude 단위 변환(5/10), P3 timemax 과소추정(5/10)
\end{itemize}

\textbf{4.2 Architecture Ablation (EXP-B, {\small\color{gray}$\sim$2p})}
\begin{itemize}[nosep,leftmargin=1.5em]
\item 실험 설계: 2$\times$2 factorial --- Domain Prompt $\times$ DualSPHysics Tools
\item B0(Full)=72.2\%, B1($-$Prompt)=0\%, B2($-$Tool)=58.3\%, B4(Bare)=0\%
\item \textbf{Prompt 주효과 +65.3\%pp $\gg$ Tool 주효과 +6.9\%pp, 상호작용 +13.9\%pp}
\item 핵심 발견 1: Prompt 없으면 모델이 DualSPHysics 도구를 \textbf{단 1회도 호출 안 함} (B1: glob/edit만 사용)
\item 핵심 발견 2: Prompt만으로도 XML 구조 생성 가능하나, 도구가 geometry 정확도를 보장 (tank\_z/fill\_height 뒤바뀜 방지)
\item 핵심 발견 3: 32B $\equiv$ 8B \textbf{모든 ablation 조건에서 동일} --- 아키텍처가 모델보다 중요
\end{itemize}

\subsection{5. Physics Validation: SPHERIC Test 10 ({\small\color{gray}$\sim$3p})}
\begin{itemize}[nosep,leftmargin=1.5em]
\item 벤치마크 설정: Souto-Iglesias (2011), 102회 반복 실험
\item M1--M8 정량 메트릭 프레임워크 정의
\item 결과: Water Lat \textbf{PASS} (M2=19.5\%, r=0.655), Oil \textbf{PASS}, Roof \textbf{PARTIAL}
\item 수렴: 3-level dp, r = 0.460 $\rightarrow$ 0.655 $\rightarrow$ 0.697
\item 기존 문헌 비교: English 2022는 동일 벤치마크에서 정량 메트릭 0개
\end{itemize}

\subsection{6. Discussion ({\small\color{gray}$\sim$3p})}

\textbf{6.1 접근성 vs 자동화}
\begin{itemize}[nosep,leftmargin=1.5em]
\item 기존 7종 에이전트: CFD 전문가의 ``자동화'' (workflow acceleration)
\item SlosimAgent: 비전문가의 ``접근'' (capability enabling, 0\% $\rightarrow$ 61\%)
\item 비전문가 baseline = 0\% (DualSPHysics XML 수동 작성 불가) $\rightarrow$ 61\% 도약이 핵심 기여
\end{itemize}

\textbf{6.2 Tool-Induced Error: 도구의 이중적 역할}
\begin{itemize}[nosep,leftmargin=1.5em]
\item B0(Full) vs B2($-$Tool) 파라미터별 비교: 3 시나리오 $\times$ 22 checks
\item \textbf{Tool HELPS} (2 params): fill\_height (3/3$\rightarrow$0/3), tank\_z (2/2$\rightarrow$0/2) --- B2는 두 값을 뒤바꿈
\item \textbf{Tool HURTS} (1 param): motion\_type (1/3$\rightarrow$3/3) --- xml\_generator가 pitch를 sway로 강제 변환
\item \textbf{Net effect: +13.6\%pp} --- 도구는 전체적으로 도움이나, 의미 영역 침범 시 오류 도입
\item 해석: Prompt = 의미(무엇을), Tool = 구조(어떻게). Tool이 ``의미'' 영역(motion\_type)까지 결정할 때 오류 발생
\end{itemize}

\textbf{6.3 Performance Ceiling: 도구 품질이 상한선}
\begin{itemize}[nosep,leftmargin=1.5em]
\item P1(motion\_type 버그) + P2(amplitude 단위 변환) 수정 시: M-A3 61\% $\rightarrow$ \textbf{75\%} (+14\%pp)
\item P3(timemax) + P4(geometry) 추가 수정 시: M-A3 $\rightarrow$ $\sim$88\%
\item 모델 스케일링(32B$\rightarrow$8B)의 영향 2.5\%pp $\ll$ 도구 버그 수정의 영향 14\%pp
\item 결론: ``더 큰 모델''이 아니라 ``더 나은 도구''가 성능 개선의 핵심 경로
\end{itemize}

\textbf{6.4 모델 크기 무관성의 함의}
\begin{itemize}[nosep,leftmargin=1.5em]
\item 32B$\approx$8B ($\Delta$=2.5\%pp), 모든 ablation 조건에서 동일
\item 8B 로컬 배포 $\rightarrow$ 엣지 디바이스/산업 현장 가능 (클라우드 의존 제거)
\item 도구가 출력 형식을 강제하므로, 모델의 ``자유도''가 제한됨 $\rightarrow$ 크기 무관
\end{itemize}

\textbf{6.5 한계 및 향후 연구}
\begin{itemize}[nosep,leftmargin=1.5em]
\item M-A3 61\% (도구 수정 시 75\%), DBC only (mDBC diverged)
\item 원통형/STL geometry 미지원 (S07, S09, S10 실패 원인)
\item Qwen3 단일 모델 패밀리 --- LLaMA/Gemma 교차 검증 필요
\item 비전문가 사용자 연구 미실시 --- NL 프롬프트 작성 능력 미검증
\item S10 8B 비결정성 ($\sigma$=35.4) --- temperature=0에서도 발생, tool calling 불안정
\end{itemize}

\subsection{7. Conclusion}
\begin{enumerate}[nosep,leftmargin=1.5em]
\item 최초 비전문가용 SPH 슬로싱 AI 에이전트 --- M-A3 61\%, 10 시나리오 60 runs
\item 최초 SPH LLM 에이전트 (DualSPHysics + Qwen3 + MCP) --- 32B$\approx$8B 로컬 배포
\item 2$\times$2 ablation: 도메인 프롬프트 +65\%pp, 도구 +7\%pp, 시너지 +14\%pp
\item SPHERIC T10 정량 검증 (M2=19.5\%, r=0.655) + M1--M8 프레임워크
\end{enumerate}

\section{Contribution Statement}
% ============================================================

\begin{quote}
\textit{To our knowledge, SlosimAgent is the first AI agent enabling non-expert technicians to configure, execute, and analyze SPH sloshing simulations through natural language. We contribute: (1) a domain-specialized agent integrating DualSPHysics GPU with a local SLM (Qwen3-32B/8B) via 18 MCP tools --- the first LLM agent for particle-based CFD; (2) E2E evaluation across 10 scenarios with 60 runs achieving 61\% M-A3 parameter fidelity, where 32B$\approx$8B ($\Delta$=2.5\%pp); (3) a 2$\times$2 factorial ablation quantifying the contribution of domain prompt (+65.3\%pp) vs.\ tools (+6.9\%pp) with +13.9\%pp interaction; and (4) quantitative validation against SPHERIC Test 10, reporting M2=19.5\% and r=0.655 with a proposed M1--M8 framework.}
\end{quote}
