\midrule

\subsection{2.2 Ablation용 Domain Prompt (\texttt{domain\_prompt.txt}, B2 조건)}

B2(도구 미제공) 조건에서는 도구 호출이 불가하므로, 프롬프트를 수정하여 LLM이 XML을 \textbf{직접 텍스트로 생성}하도록 지시한다:

\begin{quote}
\small\itshape
``사용자의 자연어 요청을 DualSPHysics v5.4 XML 케이스 파일로 직접 변환합니다. XML만 출력하세요. 설명이나 코드 블록 마크다운은 포함하지 마세요.''
\end{quote}

동일한 파라미터 자동 결정 규칙과 도메인 지식(공진 주파수, 운동 유형)을 포함하되, 도구 호출 순서 섹션은 제외하고 XML 템플릿 구조를 직접 제시한다.

\section{3. 파라미터 Fallback 체계}
% ============================================================

사용자가 값을 지정하지 않았을 때, \textbf{2단계 fallback}이 작동한다:

\subsection{3.1 1단계: LLM 추론 (System Prompt 규칙)}

LLM이 시스템 프롬프트의 ``파라미터 자동 결정 규칙''에 따라 누락 파라미터를 채운다:

\begin{center}
\small
\begin{tabular}{lll}
\toprule
\textbf{누락 파라미터} & \textbf{Fallback 규칙} & \textbf{예시} \\
\midrule
dp (파티클 간격) & min(L, W, H) / 50 & 0.6m 탱크 $\rightarrow$ dp=0.006m \\
time\_max & 5 / freq & freq=1Hz $\rightarrow$ 5s \\
fluid\_height & tank\_height $\times$ 0.5 & H=0.65m $\rightarrow$ 0.325m \\
amplitude & tank\_length $\times$ 0.05 & L=0.6m $\rightarrow$ 0.03m \\
motion\_type & ``sway'' (기본값) & 키워드 없으면 수평 병진 \\
\bottomrule
\end{tabular}
\end{center}

\subsection{3.2 2단계: 도구 Validation (\texttt{xml\_generator.go})}

LLM이 전달한 파라미터를 도구가 검증한다:

\begin{center}
\small
\begin{tabular}{ll}
\toprule
\textbf{검증 규칙} & \textbf{동작} \\
\midrule
tank\_length $\leq$ 0 & 에러 반환: ``탱크 치수는 0보다 커야 합니다'' \\
fluid\_height $>$ tank\_height & 에러 반환: ``유체 높이가 탱크 높이보다 클 수 없습니다'' \\
freq, amplitude, dp, time\_max $\leq$ 0 & 에러 반환 \\
out\_path = ``'' & 에러 반환 \\
motion\_type = ``'' & ``sway'' 기본값 적용 (코드 내 default) \\
boundary\_method $\notin$ \{dbc, mdbc\} & 에러 반환 \\
\bottomrule
\end{tabular}
\end{center}

\subsection{3.3 Fallback 실패 사례 (P3 오류 패턴)}

\texttt{time\_max} 미지정 시 프롬프트 규칙 ``5/freq''가 적용되나, ground truth보다 짧은 경우가 6/10 시나리오에서 발생:

\begin{itemize}[nosep]
\item S04: 기대 30s, 규칙 5/0.66 = 7.6s $\rightarrow$ \textbf{과소추정} (NL에 ``30초'' 명시 시 PASS)
\item S06: 기대 15s, 규칙 5/0.6 = 8.3s $\rightarrow$ 과소추정
\end{itemize}

\textbf{해석}: Fallback 규칙은 ``합리적 기본값''을 제공하나, 시나리오별 물리적 요구(정상상태 도달 시간 등)를 반영하지 못한다.

\subsection{5.2 B1 (--Prompt) 상세: 도구가 있어도 호출하지 않는 현상}

\begin{itemize}[nosep]
\item 환경변수 \texttt{SLOSIM\_GENERIC\_PROMPT=1}로 도메인 프롬프트를 비활성화
\item LLM에 15개 DualSPHysics 도구가 등록되어 있으나, \textbf{단 1회도 호출하지 않음}
\item 대신 \texttt{glob}, \texttt{edit} 등 범용 파일 도구만 사용 $\rightarrow$ XML 생성 실패
\item 40/40 runs (10 시나리오 $\times$ 2 모델 $\times$ 2 trials) 전부 M-A3 = 0\%
\item \textbf{결론}: 도메인 프롬프트가 도구 호출의 \textbf{필수조건(prerequisite)}
\end{itemize}

\subsection{5.3 B2 (--Tool) 상세: LLM 직접 XML 생성}

\begin{itemize}[nosep]
\item Ollama API 직접 호출 (\texttt{/api/generate}), tool use 비활성
\item 도메인 프롬프트(\texttt{domain\_prompt.txt})에 XML 템플릿 구조와 파라미터 규칙 포함
\item LLM이 XML 텍스트를 직접 생성 $\rightarrow$ \textbf{구조적으로 유효하나 세부 오류 존재}
\end{itemize}

\textbf{B2의 핵심 관찰}: LLM이 도구 없이 pitch(\texttt{mvrotsinu})를 \textbf{올바르게 생성}:

\begin{lstlisting}
<!-- B2_S01 (LLM 직접 생성, 도구 미사용) -->
<mvrotsinu id="1" duration="7.68">  <!-- pitch 정확! -->
  <axisp1 x="0.45" y="0.031" z="0" />
  <axisp2 x="0.45" y="-0.031" z="0" />
  <freq v="0.6515" />
  <ampl v="4" />  <!-- degrees 정확! -->
</mvrotsinu>
\end{lstlisting}

\textbf{반증거}: v3 도구(B0)는 같은 시나리오에서 mvrectsinu(sway)를 강제 생성 $\rightarrow$ \textbf{도구의 P1 한계가 성능 천장}임을 직접 증명.

\textbf{B2의 한계}: geometry 오류 (tank\_z/fill\_height 뒤바뀜), 비표준 XML 요소 (hallucination), pointmin/pointmax 불일치 $\rightarrow$ 도구가 geometry 정확도를 보장하는 역할.
