## v0.1 — MVP: E2E 최소 파이프라인

**테마:** "자연어 한 줄 → 시뮬레이션 → 결과 확인"이 동작하는 최소 제품
**예상 기간:** 4주
**상세 명세:** `docs/FRD_v0.1.md`

### 포함 기능

| FRD ID | PRD 매핑 | 기능명 | 설명 |
|--------|---------|--------|------|
| AGENT-01 | IN-01, IN-03, SIM-03 | Sloshing Coder 시스템 프롬프트 | Qwen3용 슬로싱 도메인 프롬프트 (≤8K 토큰, 한국어) |
| AGENT-02 | SIM-02 | XML 케이스 자동 생성 | 추론된 조건 → DualSPHysics XML 변환 (직사각형 탱크만) |
| TOOL-01 | SIM-01 | GenCase Tool | XML → 바이너리 파티클 전처리 (Docker) |
| TOOL-02 | SIM-01 | Solver Tool | DualSPHysics GPU 솔버 실행 (백그라운드 goroutine) |
| TOOL-03 | — | PartVTK Tool | 시뮬레이션 결과 → VTK 파일 내보내기 |
| TOOL-04 | — | MeasureTool | 특정 위치의 수위/압력 시계열 CSV 추출 |
| JOB-01 | SIM-04 | Job Manager | 단일 Job 제출/상태 조회/진행률(Part 카운팅) |
| RPT-01 | RPT-01, RPT-02 | Markdown 리포트 | 해석 조건 요약 + 수위 시계열 테이블 |
| TUI-01 | — | 상태 바 시뮬레이션 표시 | 기존 OpenCode 상태 바 활용, 진행률/완료/실패 |
| PIPE-01 | — | E2E 파이프라인 | 12단계 자동 오케스트레이션 (자연어 → 리포트) |

### v0.1 도메인 범위

| PRD ID | 기능 | v0.1 범위 |
|--------|------|----------|
| DOM-01 | 탱크 형상 | **직사각형만** |
| DOM-02 | 유체 | **물 (단일 유체)** |
| DOM-03 | 가진 조건 | **정현파만** |
| DOM-04 | 차원 | **3D만** |

### v0.1 성공 기준

```
사용자: "1m × 0.5m 직사각형 탱크에 물 50% 채우고 0.5Hz로 흔들어줘"
  → AI 조건 추론 + 사용자 확인
  → GenCase → DualSPHysics GPU → PartVTK + MeasureTool
  → report.md 생성 + 경로 안내
```

### 구현 순서

```
Week 1: AGENT-01 → AGENT-02 → TOOL-01
Week 2: TOOL-02 + JOB-01 (병렬)
Week 3: TOOL-03 + TOOL-04 (병렬) → RPT-01
Week 4: TUI-01 → PIPE-01 (통합 + QA)
```

## v0.2 — 시각화 + AI 분석

**테마:** 시뮬레이션 결과를 "보고 이해하는" 기능
**예상 기간:** 5주
**의존 근거:** v0.1에서 VTK/CSV 파일은 생성되지만 비전문가가 해석할 수 없음. 이미지/대시보드로 결과를 시각적으로 제공해야 실용적 가치 발생.

### 포함 기능

| ID | PRD 매핑 | 기능명 | 설명 |
|----|---------|--------|------|
| VIS-01 | POST-01 | pvpython 후처리 통합 | ParaView CLI Docker 통합, 렌더링 파이프라인 |
| VIS-02 | POST-02 | 정지 이미지 스냅샷 | 특정 타임스텝 렌더링 (압력/속도 컬러맵) |
| VIS-03 | POST-04 | 시각화 항목 확장 | 압력 분포, 속도장, 수위 시각화 옵션 |
| MON-01 | SIM-05 | 실시간 모니터링 | Run.csv 파싱, 에너지 수렴, 파티클 이탈 감지 |
| RPT-02 | RPT-03 | 이미지 포함 리포트 | Markdown에 스냅샷 이미지 삽입 |
| RPT-03 | RPT-04 | AI 물리적 해석 | Qwen3이 결과 데이터 기반 물리적 코멘트 생성 |
| DIM-01 | DOM-04 | 2D 시뮬레이션 지원 | 2D XML 템플릿, Y축 제거 로직 |
| TUI-02 | PRD §7 | Sim Dashboard | 전용 BubbleTea 뷰 (Job 상태, 진행률, 로그) |
| TUI-03 | PRD §7 | Result Viewer | 결과 파일 브라우저, 이미지 경로 표시 |

### 의존성 체인

```
VIS-01 (pvpython Docker)
  ├──→ VIS-02 (스냅샷) ──→ RPT-02 (이미지 리포트)
  └──→ VIS-03 (시각화 항목)
MON-01 (실시간 모니터링) ──→ TUI-02 (Sim Dashboard)
RPT-03 (AI 해석) ← Qwen3 도메인 성능 검증 (v0.1 운영 데이터 기반)
```

### 주요 기술 결정사항

| 미결 사항 (PRD §10) | v0.2 결정 |
|---------------------|----------|
| ParaView Docker 통합 vs 호스트 설치 | Docker multi-stage (pvpython 전용 이미지) |
| Qwen3 32b vs 8b 성능 비교 | v0.1 운영 데이터로 비교 후 결정 |

### 구현 순서

```
Week 1: VIS-01 (pvpython Docker 이미지 + 기본 렌더링)
Week 2: VIS-02 + VIS-03 (스냅샷 + 시각화 항목)
Week 3: MON-01 + DIM-01 (모니터링 + 2D)
Week 4: TUI-02 + TUI-03 (대시보드 + 결과 뷰어)
Week 5: RPT-02 + RPT-03 (이미지 리포트 + AI 해석) + 통합 QA
```

## v0.3 — 도메인 확장 + 멀티 Job

**테마:** 더 다양한 탱크/조건/입력, 동시 여러 시뮬레이션 실행
**예상 기간:** 5주
**의존 근거:** 시각화(v0.2)가 있어야 다양한 케이스 결과를 확인/비교 가능. 멀티 Job은 파라메트릭 스터디(v1.0)의 전제 조건.

### 포함 기능

| ID | PRD 매핑 | 기능명 | 설명 |
|----|---------|--------|------|
| GEO-01 | DOM-01 | 원통형 탱크 | 원통 좌표계 XML 지오메트리 생성 |
| GEO-02 | DOM-01 | L형 탱크 | 복합 박스 조합 지오메트리 |
| EXC-01 | DOM-03 | 지진파 가진 | 외부 시계열 파일 파싱 + motion 변환 |
| EXC-02 | DOM-03 | 임의 가진 | 사용자 정의 가진 함수 입력 |
| STL-01 | IN-02 | STL 파일 입력 | CAD 메시 임포트 + 품질 검증 |
| ANI-01 | POST-03 | 애니메이션 생성 | pvpython + ffmpeg → mp4/gif |
| JOB-02 | SIM-04 확장 | 동시 다중 Job 관리 | Job 큐, 동시 실행 제한, 우선순위 |
| TUI-04 | PRD §7 | Case Wizard | 대화형 단계별 조건 설정 UI (BubbleTea) |

### 의존성 체인

```
GEO-01/GEO-02 (새 지오메트리) ← AGENT-02 XML 생성 로직 확장
EXC-01/EXC-02 (새 가진) ← motion 섹션 XML 생성 확장
STL-01 (STL 입력) ← 별도 메시 변환 Tool 필요
ANI-01 (애니메이션) ← VIS-01 (pvpython, v0.2)
JOB-02 (멀티 Job) ← JOB-01 (v0.1) + MON-01 (v0.2)
TUI-04 (Case Wizard) ← GEO-01/02 + EXC-01/02 (다양한 옵션 필요 시 유용)
```

### 주요 기술 결정사항

| 미결 사항 (PRD §10) | v0.3 결정 |
|---------------------|----------|
| STL 입력 시 메시 품질 검증 로직 | 수밀성 검사 + 법선 방향 검증 (DualSPHysics GenCase 에러 메시지 파싱) |

### 구현 순서

```
Week 1: GEO-01 (원통형) + EXC-01 (지진파) — XML 생성 로직 확장
Week 2: GEO-02 (L형) + EXC-02 (임의 가진)
Week 3: STL-01 (STL 입력 + 메시 검증)
Week 4: JOB-02 (멀티 Job) + ANI-01 (애니메이션)
Week 5: TUI-04 (Case Wizard) + 통합 QA
```

## v1.0 — 파라메트릭 스터디 + 프로덕션

**테마:** 전문가 수준 활용, 비교 분석, 프로덕션 안정성
**예상 기간:** 4주
**의존 근거:** 파라메트릭 스터디는 멀티 Job(v0.3) + 시각화(v0.2) + 다양한 도메인(v0.3)이 **모두** 필요. 여러 케이스를 동시 실행하고, 결과를 시각적으로 비교하고, 리포트로 요약해야 하므로 이전 버전의 모든 기능이 전제.

### 포함 기능

| ID | PRD 매핑 | 기능명 | 설명 |
|----|---------|--------|------|
| PARA-01 | PARA-01 | 파라메트릭 스터디 실행 | "충진율 50/60/70% 비교" → 자동 3건 실행 |
| PARA-02 | PARA-02 | 이전 결과 비교 | 시뮬레이션 이력에서 선택하여 비교 |
| PARA-03 | PARA-03 | 비교 리포트 생성 | 비교 테이블 + 오버레이 시각화 |
| STORE-01 | PRD §7 | Result Store (DB) | SQLite 기반 시뮬레이션 이력 관리 |
| TUI-05 | PRD §7 | Parametric View | 비교 테이블 + 결과 오버레이 뷰 |
| NFR-01 | PRD §8 | 에러 처리 고도화 | 발산/에러 자동 감지 + 재시도/복구 전략 |
| NFR-02 | PRD §8 | 확장성: 템플릿 플러그인 | 새 케이스 템플릿을 파일 드롭으로 추가 |
| NFR-03 | PRD §8 | 데이터 보존 | 모든 시뮬레이션 결과 + 리포트 이력 관리 |
| DOC-01 | — | 사용자 문서 + 온보딩 | 설치 가이드, 튜토리얼, FAQ |

### 의존성 체인

```
PARA-01 (파라메트릭 실행)
  ← JOB-02 (멀티 Job, v0.3) — 동시 다중 시뮬레이션 실행
  ← VIS-01~03 (시각화, v0.2) — 결과 비교 시 이미지 필요
  ← GEO-01/02, EXC-01/02 (v0.3) — 다양한 파라미터 조합

PARA-02 (결과 비교) ← STORE-01 (Result Store)
PARA-03 (비교 리포트) ← PARA-01 + PARA-02 + RPT-02 (이미지 리포트, v0.2)
TUI-05 (Parametric View) ← TUI-02 (Dashboard, v0.2) + PARA-01
```

### 주요 기술 결정사항

| 미결 사항 (PRD §10) | v1.0 결정 |
|---------------------|----------|
| 결과 저장소 구조 (DB vs 파일시스템) | 하이브리드: SQLite 메타데이터 + 파일시스템 원본 데이터 |

### 구현 순서

```
Week 1: STORE-01 (Result Store DB 스키마 + 마이그레이션)
Week 2: PARA-01 (파라메트릭 실행 엔진) + PARA-02 (결과 비교 로직)
Week 3: PARA-03 (비교 리포트) + TUI-05 (Parametric View)
Week 4: NFR-01~03 (안정성 + 확장성) + DOC-01 (문서) + 전체 QA
```
